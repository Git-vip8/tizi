服务器被重置后，从头部署的完整步骤和命令如下：

---

第一步：SSH 登录服务器

```bash
ssh root@你的新IP
```

第二步：克隆你的 GitHub 仓库

```bash
git clone https://github.com/Git-vip8/tizi.git
cd tizi
```

第三步：执行部署脚本

```bash
bash bushu.sh
```

第四步：验证两个节点都在运行

```bash
ps aux | grep sing-box
```

---

就这么简单。 重置后不需要重新生成密钥，不需要重新写配置，因为你的仓库里已经有了完整的配置文件、核心程序压缩包，以及自动生成证书并启动节点的部署脚本。一条龙恢复。

唯一需要注意：如果换服务器导致 IP 变了，你需要更新仓库里两个配置文件中的 IP 地址（43.131.225.218 改成新IP），以及重新生成 VLESS 的密钥对（因为 Reality 密钥和 IP 绑定）。需要我把“换IP后要改哪些配置”也整理出来吗？



服务器重启后，之前运行的节点进程会被停掉，需要手动重新启动。执行：

```bash
cd ~/tizi && bash bushu.sh
```
